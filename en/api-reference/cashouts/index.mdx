A **Cashout** represents a withdrawal request that transfers funds from your Safefy balance to an external PIX key.

## The Cashout object

```json
{
  "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "externalId": "withdrawal-001",
  "amount": 5000,
  "fee": 100,
  "netAmount": 4900,
  "currency": "BRL",
  "status": "Completed",
  "environment": "Production",
  "pix": {
    "pixKeyType": "CPF",
    "pixKey": "12345678901",
    "endToEndId": "E1234567890"
  },
  "requestedAt": "2024-01-15T10:00:00Z",
  "processedAt": "2024-01-15T10:01:00Z",
  "completedAt": "2024-01-15T10:02:00Z",
  "failureReason": null,
  "createdAt": "2024-01-15T10:00:00Z"
}
```

---

## Attributes

<ParamField body="id" type="string">
  Unique UUID v4 identifier, auto-generated by Safefy.
</ParamField>

<ParamField body="externalId" type="string">
  Your internal identifier for this cashout.
</ParamField>

<ParamField body="amount" type="integer">
  Withdrawal amount in **cents**. Example: `5000` = R$50.00.
</ParamField>

<ParamField body="fee" type="integer">
  Safefy platform fee for this cashout, in cents.
</ParamField>

<ParamField body="netAmount" type="integer">
  Amount after fee deduction (`amount - fee`), in cents. This is the amount received at the destination.
</ParamField>

<ParamField body="currency" type="string">
  Currency code. Always `BRL`.
</ParamField>

<ParamField body="status" type="string">
  Current cashout status. See the [lifecycle](#cashout-lifecycle) section below.
</ParamField>

<ParamField body="environment" type="string">
  Environment where this cashout was created: `Sandbox` or `Production`.
</ParamField>

<Expandable title="pix">
  <ParamField body="pix.pixKeyType" type="string">
    Type of PIX key used: `CPF`, `CNPJ`, `Email`, `Phone`, or `Random`.
  </ParamField>
  <ParamField body="pix.pixKey" type="string">
    The destination PIX key.
  </ParamField>
  <ParamField body="pix.endToEndId" type="string">
    End-to-end transaction ID assigned by the financial system. Filled after processing.
  </ParamField>
</Expandable>

<ParamField body="requestedAt" type="string">
  Date and time when the cashout was requested, in ISO 8601 format.
</ParamField>

<ParamField body="processedAt" type="string">
  Date and time when processing began, in ISO 8601 format.
</ParamField>

<ParamField body="completedAt" type="string">
  Date and time when the transfer was completed, in ISO 8601 format.
</ParamField>

<ParamField body="failureReason" type="string">
  Reason for failure or rejection. Only populated when status is `Failed` or `Rejected`.
</ParamField>

<ParamField body="createdAt" type="string">
  Date and time when the record was created, in ISO 8601 format.
</ParamField>

## Cashout lifecycle

```
Pending  Processing  Completed
               
Cancelled     Failed
                
            Rejected
```

| Status | Description |
|--------|-------------|
| `Pending` | Cashout requested, awaiting processing. Can still be cancelled. |
| `Processing` | Transfer in progress with the acquirer |
| `Completed` | Transfer completed successfully |
| `Cancelled` | Cashout was cancelled before processing began |
| `Failed` | An error occurred during processing |
| `Rejected` | Transfer rejected (invalid PIX key, non-existent account, etc.) |

---

## PIX key types

| Type | Format |
|------|--------|
| `CPF` | 11 digits (numbers only) |
| `CNPJ` | 14 digits (numbers only) |
| `Email` | Valid email address |
| `Phone` | `+55` + area code + number |
| `Random` | 32 alphanumeric characters |
